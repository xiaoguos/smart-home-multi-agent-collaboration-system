# 数据挖掘 Agent 日志说明

## 📋 概述

数据挖掘 Agent 现在会在终端输出详细的日志信息，帮助您实时查看每次数据挖掘的过程和结果。

## 🎯 日志内容

### 1. 数据挖掘开始

```
================================================================================
🔍 【数据挖掘开始】用户习惯分析
================================================================================
👤 用户ID: 1000000001
📅 查询时间范围: 最近 30 天
📊 查询到 156 条设备操作记录
🎯 用户查询: 晚上回家的时候需要开什么设备
```

**说明**：
- 显示用户ID和查询时间范围
- 显示从数据库查询到的记录总数
- 显示用户的原始查询内容

---

### 2. 准备聚类数据

```
📦 【准备聚类数据】
   ✓ 特征矩阵维度: (156, 8)
   ✓ 特征列表: hour, minute, day_of_week, is_weekend, is_morning, is_afternoon, is_evening, is_night
```

**说明**：
- 特征矩阵维度：(样本数, 特征数)
- 列出用于聚类的所有时间特征

---

### 3. GMM 聚类分析

```
🤖 【GMM聚类分析】
   ℹ️  样本数量: 156
   ℹ️  目标场景数: 5
   ⏳ 正在执行高斯混合模型聚类...
   ✓ 聚类完成！识别出 5 个使用场景

**说明**：
- 显示样本数量和目标场景数
- 场景数根据数据量动态调整（min(5, 样本数//5)，至少2个）
- 显示实际识别出的场景数量

---

### 4. 场景模式分析

```
📈 【场景模式分析】

   🎬 场景 #1: 早上 
      📍 时间段: 早上 (平均 7.5:00)
      📊 出现次数: 32 次
      🔧 设备操作:

           - turn_on (temperature=22, mode=cool)
           - set_temperature (temperature=23)
         • air_cleaner:
           - turn_on (mode=auto)
           ... 还有 2 个操作

   🎬 场景 #2: 晚上 
      📍 时间段: 晚上 (平均 19.3:00)
      📊 出现次数: 45 次
      🔧 设备操作:
         • bedside_lamp:

  - **出现次数**：该场景被记录的次数
  - **设备操作**：该场景中常见的设备操作（最多显示3个，超过显示省略提示）

## 📝 用户反馈日志

当用户提交反馈时，会显示：

```
================================================================================
📝 【用户反馈处理】
================================================================================
👤 用户ID: 1000000001
🔖 会话ID: session-abc123
🎬 场景: 晚上场景 (晚上)

📊 反馈类型: modified

🔄 用户修改详情:
   1. bedside_lamp:
      原始: turn_on {'brightness': 80, 'color_temp': 3000}
      修改: turn_on {'brightness': 60, 'color_temp': 2700}
   2. air_conditioner:
      原始: turn_on {'temperature': 24}
      修改: turn_on {'temperature': 26}

💾 正在保存反馈到数据库...
   ✓ 反馈已保存

🎯 正在更新置信度模型...
   ✓ 置信度已更新

================================================================================
✅ 【用户反馈处理完成】
   反馈类型: modified
   学习状态: 已更新
================================================================================
```

**说明**：
- 显示反馈类型：`modified`（修改）或 `accepted`（接受）
- 如果用户修改了推荐，显示每个操作的原始值和修改后的值
- 显示保存和模型更新的状态

---

## 🚀 如何查看日志

### 方法1：在终端运行 Agent

```bash
cd agents/data_mining_agent
python -m data_mining_agent
```

所有日志会直接显示在终端。

### 方法2：查看 Python 日志（如果使用日志文件）

如果配置了日志文件，可以使用：

```bash
tail -f logs/data_mining_agent.log
```

## 🎨 日志符号说明

| 符号 | 含义 |
|------|------|
| 🔍 | 数据挖掘开始 |
| 📦 | 数据准备 |
| 🤖 | 机器学习（GMM） |
| 📈 | 场景分析 |
| 🎬 | 识别的场景 |
| 🎯 | 场景匹配 |
| 💡 | 生成推荐 |
| 📝 | 用户反馈 |
| ✅ | 成功完成 |
| ❌ | 错误/失败 |
| ⚠️ | 警告 |
| 🟢 | 高置信度 (>0.7) |
| 🟡 | 中置信度 (0.5-0.7) |
| 🔴 | 低置信度 (<0.5) |

## 💡 提示

1. **性能影响**：这些日志主要用于开发和调试，生产环境可以通过调整日志级别来控制输出量
2. **颜色支持**：某些终端支持彩色 emoji，显示效果更佳
3. **日志持久化**：建议配置日志文件，以便后续分析和审计

## 🔧 配置日志级别

如果想要更详细的调试信息，可以在启动 Agent 时设置环境变量：

```bash
export LOG_LEVEL=DEBUG
python -m data_mining_agent
```

这将显示更多的内部处理细节。
