# 🚀 Moss AI 聊天系统 - 快速开始指南

## 📋 系统要求

- Python 3.8+
- Node.js 16+
- Cangjie 编译器
- 稳定的网络连接（用于调用 DeepSeek API）

## 🔧 一键安装依赖

### Windows 用户

```powershell
# 在项目根目录运行
.\script\install_dependencies.ps1
```

### 手动安装

```bash
# 1. 安装 Python 依赖
pip install -r requirements.txt

# 2. 安装前端依赖
cd app
pnpm install  # 或 npm install
cd ..

# 3. 编译 Cangjie 后端
cd app/backend
cjpm build
cd ../..
```

## 🚀 启动服务

### 方式一：使用一键启动脚本（推荐）

```powershell
# Windows
.\script\start_all_services.ps1
```

这将自动启动所有三个服务：
1. Conductor Agent API (端口 12001)
2. Cangjie 后端 (端口 2100)
3. 前端开发服务器 (端口 5173)

### 方式二：手动启动

#### 步骤 1：启动 Conductor Agent (A2A Server)
```bash
cd agents/conductor_agent
python main.py --host localhost --port 12000
```
✅ 服务运行在 `http://localhost:12000`

#### 步骤 2：启动 Cangjie 后端
```bash
cd app/backend

# Windows
dist\release\bin\main.exe

# Linux/Mac
./dist/release/bin/main
```
✅ 服务运行在 `http://127.0.0.1:2100`

#### 步骤 3：启动前端
```bash
cd app
pnpm dev  # 或 npm run dev
```
✅ 服务运行在 `http://localhost:5173`

## 🔍 故障排查

### 问题 1：前端无法连接后端
**错误信息**：`Failed to fetch` 或 `Network Error`

**解决方案**：
1. 确认后端服务正在运行：`curl http://127.0.0.1:2100/hello`
2. 检查端口是否被占用
3. 查看浏览器控制台的详细错误

### 问题 2：后端无法连接 Conductor Agent
**错误信息**：`无法连接到 Conductor Agent`

**解决方案**：
1. 确认 Conductor Agent 正在运行：`curl http://localhost:12000/`
2. 检查 Python 依赖是否完整安装（特别是 a2a-sdk）
3. 查看 Conductor Agent 的日志输出

### 问题 3：AI 无响应或响应错误
**可能原因**：DeepSeek API 配置问题

**解决方案**：
1. 检查 `agents/conductor_agent/agent.py` 中的 API Key 是否有效
2. 确认网络连接正常
3. 查看 Conductor Agent 日志中的详细错误信息

### 问题 4：A2A 协议错误
**错误信息**：响应格式不正确

**解决方案**：
1. 确保 A2A SDK 版本正确
2. 检查请求的消息格式是否符合 A2A 协议规范
3. 查看 Cangjie 后端日志中的详细错误信息
