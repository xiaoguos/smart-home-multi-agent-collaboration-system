# 局域网配置说明

## 📝 配置步骤

### 1. 创建环境配置文件

在 `app` 目录下创建 `.env.development` 文件：

```bash
cd app
# 复制示例文件
copy .env.example .env.development
```

### 2. 修改配置

编辑 `.env.development`，只需修改 IP 地址：

```env
# 后端地址（只需修改 IP 即可，端口固定）
VITE_BACKEND_URL=http://192.168.1.100:2100
```

**说明：**
- 端口 2100 是 Cangjie 后端固定端口
- 端口 12000 是 Conductor Agent 固定端口
- 只需要修改 IP 地址即可

### 3. 代理工作原理

```
前端 (本机)
    ↓ /api/chat
Vite 代理
    ↓ http://192.168.1.100:2100/api/chat
后端服务器 (局域网)
    ↓
Conductor Agent (局域网)
```

## 🚀 启动方式

### 本机（前端）
```bash
cd app
pnpm dev
# 访问 http://localhost:1420
```

### 局域网服务器（后端 + Agent）
```bash
# 启动 Conductor Agent
cd agents/conductor_agent
python main.py --host 0.0.0.0 --port 12000

# 启动 Cangjie 后端
cd app/backend
dist\release\bin\main.exe
```

⚠️ **注意**：后端服务器需要绑定到 `0.0.0.0` 而不是 `127.0.0.1`，才能接受局域网连接。

## 🔧 修改后端监听地址

如果后端 Cangjie 服务器只监听 `127.0.0.1`，需要修改：

**app/backend/src/server/default.cj**
```cangjie
// 修改前
let server = ServerBuilder().addr("127.0.0.1").port(2100).build()

// 修改后（允许局域网访问）
let server = ServerBuilder().addr("0.0.0.0").port(2100).build()
```

## 🧪 测试连接

从本机测试局域网服务器：

```bash
# 测试后端
curl http://192.168.1.100:2100/hello

# 测试 Conductor Agent
curl http://192.168.1.100:12000/
```
